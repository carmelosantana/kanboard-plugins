# CalendarPlugin v1.2.0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `calendarEventSources` container hook (plugins CONTRIBUTE events) and FullCalendar week/day time-grid views to CalendarPlugin, releasing as v1.2.0.

**Architecture:** Two independent changes. (1) `CalendarQueryModel::getEvents` merges events returned by each callable in `container['calendarEventSources']` after the existing due-date build + decorator loop, read defensively so an absent key is byte-identical to 1.1.0. (2) `Assets/js/calendar.js` adds `timeGridWeek`/`timeGridDay` to the FullCalendar view set and header toolbar, with resize explicitly disabled; no backend/events-feed change (the feed already parses `start`/`end`). CalendarPlugin registers NO source of its own.

**Tech Stack:** PHP 8.4, Kanboard core v1.2.47 (PicoDb container, hooks), FullCalendar v6.1.15 global build (buildless, route-gated), PHPUnit (in-memory SQLite via `testing/run-plugin-tests.sh`).

```php
        // Generic extension point: other suite plugins (e.g. TimeBlock) may
        // register SOURCES on the container to CONTRIBUTE new events (distinct
        // from decorators, which only mutate the due-date events above). Each
        // source is a callable fn($userId, $filters, $rangeStart, $rangeEnd): array
        // returning fully-formed FullCalendar events (namespaced string ids).
        // The source owns access control, range-filtering, and its own badges —
        // CalendarPlugin appends them verbatim. Read defensively: absent key ⇒
        // [] ⇒ behavior byte-identical to before this extension point.
        $sources = isset($this->container['calendarEventSources']) ? $this->container['calendarEventSources'] : array();
        foreach ($sources as $source) {
            foreach (call_user_func($source, $userId, $filters, $rangeStart, $rangeEnd) as $ev) {
                $events[] = $ev;
            }
        }

        return $events;
```

## Cross-plugin: `calendarEventSources`

Where `calendarEventDecorators` (above) lets a plugin *mutate* the due-date
events CalendarPlugin already builds, `calendarEventSources` lets a plugin
*contribute new* events. Any plugin can, in its own `initialize()`, append a
callable to a container-registered array:

```php
$this->container['calendarEventSources'] = array_merge(
    isset($this->container['calendarEventSources']) ? $this->container['calendarEventSources'] : array(),
    array(
        function (int $userId, array $filters, int $rangeStart, int $rangeEnd): array {
            // Query your own data, build FullCalendar events for this
            // user/range, and return them. Return [] if nothing applies.
            return array();
        },
    )
);
```

Contract:

- `calendarEventSources` is an **optional** array of callables with signature
  `function(int $userId, array $filters, int $rangeStart, int $rangeEnd): array`.
  `$filters` is the same array `CalendarController::events()` builds
  (`project_ids`, `assignee_id`, `category_id`, `column_id`, `hide_completed`);
  `$rangeStart`/`$rangeEnd` are unix timestamps for the visible range.
- Each callable returns a list of FullCalendar event arrays with the same shape
  as the built-in due-date events (`id`, `title`, `start`, `end`, `allDay`,
  `color`, `url`, `extendedProps`).
- **The source owns access control** — it MUST filter to projects `$userId` may
  access and honor `$filters` where meaningful. CalendarPlugin does NOT
  re-check source events.
- **Only return events overlapping `[$rangeStart, $rangeEnd)`.**
- **`id` MUST be namespaced** (e.g. `timeblock-<taskId>`) — unique across
  sources and distinct from due-date event ids (which are the int task id).
- **Decorators are NOT applied to source events** — a source builds complete
  events itself, including any `extendedProps.badges`.
- CalendarPlugin reads the container entry **defensively**, so with no sources
  registered the calendar renders identically to before this extension point.

```
